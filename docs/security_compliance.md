# Security and Compliance Guide

## Data Protection
- Customer data is encrypted at rest using AES-256.
- Data in transit is protected using TLS 1.3.
- Customer data is not sold to third parties.

## Compliance
- Savannah Tech Innovations is committed to compliance with the Zimbabwe Cyber and Data Protection Act.
- Security practices should be reviewed regularly as laws and product requirements change.

## Access and Secrets
- API keys should be kept private.
- Keys should never be committed into source code or exposed in frontend apps.
- Access should be restricted to authorized team members.

## Auditing and Review
- Security audits are conducted quarterly by independent cybersecurity firms.
- Teams should review access logs, incident logs, and integration logs regularly.

## Backup and Retention
- Customers should confirm backup schedules for production systems.
- Data retention policies may vary by product and contract.
- Enterprise customers may request custom retention rules.
